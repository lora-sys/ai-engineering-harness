# Real Cases — index

These three cases walk `$build-agent-app` end-to-end against three different entry workflows. They're from the v1.0.1 release and represent the **calibration set** for the skill: they show what good looks like on a real-shaped input.

> **Read these when:**
> - You want to verify the skill's contract before integrating it elsewhere.
> - You're an evaluator / reviewer and want a concrete reference rather than rule-of-thumb.
> - You're a maintainer and want to see the patterns the skill defends.

## What this is not

- These are not "best practice" canons. They are reference instances. A different input would produce different contracts.
- These are not exhaustive. They cover three inputs that the maintainer considered representative.
- These are not stable. When `templates/` or `references/` change, expect the corresponding outputs to be regenerated.

## Reproducing / re-generating

Each case's README ends with a one-line `Use $build-agent-app …` invocation. Save that into a Codex session to regenerate the case from scratch.
